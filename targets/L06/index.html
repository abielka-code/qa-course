About / API Specification (спрощені вимоги)


      
Base

      

        Усі запити виконуються до умовного API, яке починається з /api.
        Це симулятор API всередині сторінки (без реального сервера).
      


      
Endpoints

      
    
        
  • GET /api/products → 200 OK, JSON-масив товарів
    • 
        
  • GET /api/products/:id → 200 OK або 404 Not Found
    • 
        
  • POST /api/products → 201 Created (створення товару)
    • 
        
  • PUT /api/products/:id → 200 OK або 404 Not Found
    • 
        
  • DELETE /api/products/:id → 204 No Content або 404 Not Found
    • 
        
  • POST /api/login → 200 OK або 401 Unauthorized
    • 
      


      
Product schema

      
    
        
  • id — number
    • 
        
  • name — string (обов’язкове)
    • 
        
  • price — number (обов’язкове, ≥ 0)
    • 
      


      
Login schema

      
    
        
  • Body: {"username":"admin","password":"1234"}
    • 
        
  • Коректні дані: admin / 1234
    • 
      


      
Важливо

      

        Потрібно не “шукати баги випадково”, а виконати чіткі HTTP-запити,
        проаналізувати статус-коди та відповіді й зафіксувати їх у звіті.
