About / Scenario Rules

      
    
        
  • Сценарій — це послідовність API запитів (кроків), об’єднаних спільною метою.
    • 
        
  • У сценарії важливо визначити контрольні точки: статус-коди та дані відповіді.
    • 
        
  • Після кроків Create/Update потрібно перевіряти, що зміни відобразилися через Read (GET).
    • 
        
  • Після Delete потрібно перевірити, що ресурс недоступний (404 Not Found).
    •
