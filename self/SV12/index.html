Тема: CI/CD для тестувальника — pipeline, smoke/regression, контроль якості релізу

Роль студента: QA/Release Gatekeeper. Код не пишемо — виконуємо smoke/regression, аналізуємо стабільність релізу, збираємо докази DevTools, формуємо рішення Go/No-Go.

Мета

Освоїти підхід QA у CI/CD: що таке pipeline, які тести запускаються на етапах, як виконати smoke та regression перевірки перед релізом, як оцінити ризики та оформити release quality report.

Об’єкт тестування

Target-сайт: targets/SV12 — “ReleaseGate: релізний контроль (Auth + Orders + Status)”.

Що потрібно здати

QA-артефакти релізу
  • Smoke checklist (мінімум 15 пунктів)
  • Regression checklist (мінімум 10 пунктів)
  • Тест-кейси (8–12) зі статусами Pass/Fail/Blocked
  • Баг-репорти (4–6) з severity/priority
  • Test Summary Report + Go/No-Go
  • Release Notes (коротко: що перевірено, що критично)
Evidence
  • 10+ скріншотів UI
  • Network 404/500
  • Console errors після ключових дій
  • Скрін “реліз заблоковано / No-Go”

Ключові поняття

  • Pipeline: етапи build → test → deploy
  • Smoke: “чи живий продукт” (основні сценарії)
  • Regression: “чи не зламали старе”
  • Release gate: рішення Go/No-Go
