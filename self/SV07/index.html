Самостійна робота SV07

Тема: Аналіз дефектів (класифікація, severity/priority, root cause, рекомендації)

Роль студента: тестувальник. Код не пишемо — лише виявляємо та аналізуємо дефекти, оформлюємо звітність.

Мета

Навчитися виконувати аналіз дефектів: знаходити баги, класифікувати їх за типами, визначати severity та priority, формувати припущення про root cause та пропонувати рекомендації для виправлення. Докази збирати через UI + DevTools.

Об’єкт тестування

Target-сайт: targets/SV07 — “OrderLite: міні-замовлення (кошик + доставка)”.

Відкрити target-сайт SV07 Повернутися на головну

Що потрібно здати

  • Чек-лист (20–25+ пунктів) — включно з аналізом дефектів
  • Тест-кейси (8–12)
  • Баг-репорти (4–6 або всі знайдені)
  • Defect Analysis Sheet (таблиця аналізу дефектів: тип, severity, причина, рекомендація)
  • Test Summary Report з Go/No-Go
  • Evidence: 10+ скріншотів (UI + Console + Network + Responsive)

Підказка

  • Кожен дефект: що саме, де, як повторити, чому критично
  • Додайте root cause hypothesis: неправильна формула, відсутня перевірка, неправильний статус
  • Додайте recommendation: що змінити у логіці/UI/API
