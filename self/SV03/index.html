Самостійна робота SV03

Тема: Написання тест-кейсів для веб-застосунку

Роль студента: тестувальник. Код не пишемо — лише тестуємо, документуємо та формуємо тестову документацію.

Мета

Навчитися створювати набір тест-кейсів (8–12) для перевірки функціоналу веб-застосунку: додавання задач, зміна статусів, фільтрація, пошук, очищення, а також перевірки DevTools (Console/Network), адаптивності та базової доступності.

Об’єкт тестування

Target-сайт: targets/SV03 — “TaskFlow: менеджер задач + фільтри + синхронізація”.

Відкрити target-сайт SV03 Повернутися на головну

Що потрібно здати

  • Тест-кейси (8–12) у таблиці (ID, Preconditions, Steps, Expected Result)
  • Чек-лист (20–25+ пунктів) — як підтримка покриття
  • Баг-репорти (4–6 або всі знайдені)
  • Test Summary Report з рішенням Go/No-Go
  • Evidence: 10+ скріншотів (UI + Console + Network + Responsive)

Підказка для SV03

Тест-кейс — це не просто “перевірка”, а детальний сценарій з кроками та очікуваним результатом.

  • Не змішуйте в одному кейсі 5 перевірок
  • Додавайте Preconditions (якщо потрібні)
  • Для DevTools робіть окремі кейси
