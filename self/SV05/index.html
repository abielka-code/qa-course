Самостійна робота SV05

Тема: Аналіз HTTP-запитів (DevTools Network + Console)

Роль студента: тестувальник. Код не пишемо — лише перевірка запитів, статусів, заголовків та документування дефектів.

Мета

Навчитися аналізувати HTTP-запити у браузері через DevTools → Network: метод (GET/POST), URL, статус-код, заголовки, body (payload), відповідь (response), та підтверджувати дефекти доказами (скрін UI/Console/Network).

Об’єкт тестування

Target-сайт: targets/SV05 — “HTTP Lab: панель API-запитів”.

Відкрити target-сайт SV05 Повернутися на головну

Що потрібно здати

  • Чек-лист (20–25+ пунктів) з Network-перевірками
  • Тест-кейси (8–12), включно з DevTools кейсами
  • Баг-репорти (4–6 або всі знайдені)
  • Test Summary Report (Go/No-Go)
  • Evidence: 10+ скріншотів (UI + Console + Network)

Мінімальні докази

  • Network: 4+ скріни зі статусами (200/404 та ін.)
  • Headers: 2+ скріни (Request/Response Headers)
  • Console: 2+ скріни з помилками (якщо є)
  • UI: 2+ скріни станів “success/error”
