Самостійна робота SV04

Тема: Негативні сценарії тестування (валідація, помилки, крайні значення)

Роль студента: тестувальник. Код не пишемо — лише перевірка, аналіз та документування дефектів.

Мета

Навчитися формувати і виконувати негативні сценарії тестування для веб-форми (порожні значення, некоректні формати, граничні значення), а також підтверджувати дефекти через DevTools (Console/Network) і перевіряти адаптивність.

Об’єкт тестування

Target-сайт: targets/SV04 — “QuickApply: заявка на подію (реєстрація)”.

Відкрити target-сайт SV04 Повернутися на головну

Що потрібно здати

  • Негативні тест-кейси (8–12) — окремо від позитивних
  • Чек-лист (20–25+ пунктів) з негативними перевірками
  • Баг-репорти (4–6 або всі знайдені)
  • Test Summary Report з рішенням Go/No-Go
  • Evidence: 10+ скріншотів (UI + Console + Network + Responsive)

Підказка

Негативний сценарій — це перевірка поведінки системи у випадку помилки користувача або неочікуваних даних.

  • Порожнє поле
  • Неправильний формат (email/телефон)
  • Надто коротке/довге значення
  • Спецсимволи / пробіли / “0”
  • Перевірки через Console/Network
