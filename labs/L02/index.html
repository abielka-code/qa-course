Мета

Навчитися планувати тестування веб-застосунку: визначати обсяг (scope), типи тестування, критерії входу/виходу, ризики, тестове середовище, ресурси та артефакти. Результат роботи — документ Test Plan.

Навчальний сайт для аналізу

У цій лабораторній необхідно НЕ тестувати “все підряд”, а виконати аналіз системи та скласти план тестування.

Відкрити сайт для планування (ЛР2)

Що потрібно зробити

  1. Відкрити навчальний сайт (Test Target — ЛР2) та прочитати блок “About / Requirements”.
  2. Визначити модулі, які підлягають тестуванню.
  3. Скласти документ Test Plan за рекомендованою структурою.
  4. Описати scope: що тестуємо, що не тестуємо (In Scope / Out of Scope).
  5. Описати підхід до тестування та ключові ризики.

  • Документ: Test Plan (ЛР2) (Google Docs / DOCX / PDF)
  • Обсяг: не менше 2–4 сторінок тексту (без титульної)
