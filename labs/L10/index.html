Мета

Виконати підсумкове тестування реліз-кандидата (Release Candidate) на основі smoke-набору та вибіркового regression-набору, зафіксувати результати, сформувати Test Summary Report та прийняти рішення щодо релізу (Release decision).

Навчальний ресурс

У роботі використовується “Release Candidate Test Center” — сторінка, яка містить опис релізу, набір тестів, перелік дефектів і прості інструменти для фіксації результатів. Студенти не пишуть код, а виконують перевірки та складають підсумковий звіт.

Відкрити Release Candidate Test Center (ЛР10)

Що потрібно зробити

  1. Ознайомитися з описом релізу та областю тестування (Scope).
  2. Виконати smoke-набір (мінімум 8 тестів) та regression-набір (мінімум 6 тестів).
  3. Зафіксувати результати тестів у таблиці (Pass/Fail/Blocked/Skipped).
  4. Оформити мінімум 3 баг-репорти для Fail-тестів (якщо Fail є).
  5. Скласти Test Summary Report та сформувати Release decision.

Що здати

  • Звіт ЛР10 (Test Summary Report)
  • Таблиця тестів (smoke+regression)
  • Мінімум 3 баг-репорти (за потреби)
  • Скріншоти (мінімум 10)
  • Назва файлу: Група_Прізвище_ЛР10.pdf

Шаблон Test Summary Report

1. Release інформація (версія, дата, середовище)
2. Scope тестування (що тестували)
3. Test execution summary (Total/Pass/Fail/Blocked/Skipped)
4. Key findings (основні проблеми)
5. Defects summary (Critical/Major/Minor)
6. Risks та обмеження
7. Release decision (Go/No-Go) + обґрунтування
8. Рекомендації
