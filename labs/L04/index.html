Мета

Навчитися створювати тест-кейси для веб-застосунку: формулювати передумови, кроки, тестові дані, очікуваний результат, фактичний результат та статус виконання.

Навчальний сайт для тестування

У даній роботі використовується окремий сайт “Task Board”, що дозволяє створювати задачі, змінювати їх статуси, виконувати пошук і фільтрацію.

Відкрити сайт для тестування (ЛР4)

Що потрібно зробити

  1. Відкрити Test Target — ЛР4 та прочитати блок “About / Requirements”.
  2. Створити 10 тест-кейсів (мінімум), які покривають ключовий функціонал.
  3. Додати позитивні та негативні сценарії.
  4. Виконати тест-кейси та заповнити фактичний результат (Pass/Fail).
  5. Додати скріншоти виконання (мінімум 6).

Що здати

  • Звіт ЛР4 (Google Docs / DOCX / PDF)
  • Тест-кейси 10+ з виконанням (Pass/Fail)
  • Скріншоти 6+
  • Назва файлу: Група_Прізвище_ЛР04.pdf
