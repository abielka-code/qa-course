Мета

Навчитися складати чек-лист тестування для веб-застосунку на основі вимог, виконувати ручні перевірки та фіксувати результати (Pass/Fail).

Навчальний сайт для тестування

Для цієї лабораторної використовується окремий тестовий сайт з формою реєстрації, профілем користувача та пошуком у списку користувачів.

Відкрити сайт для тестування (ЛР3)

Що потрібно зробити

  1. Відкрити Test Target — ЛР3 та прочитати блок “About / Requirements”.
  2. Скласти чек-лист (мінімум 25 перевірок) для модулів: Registration, Profile, Users list, Search.
  3. Додати позитивні та негативні перевірки (валідація полів, межі значень, порожні значення).
  4. Виконати перевірки за чек-листом та заповнити Pass/Fail і фактичний результат.
  5. Додати скріншоти (мінімум 6).

Що здати

  • Звіт ЛР3 (Google Docs / DOCX / PDF)
  • Чек-лист 25+ пунктів із Pass/Fail
  • Скріншоти 6+
  • Назва файлу: Група_Прізвище_ЛР03.pdf
