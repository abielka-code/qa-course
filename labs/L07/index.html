Лабораторна робота №7

Розробка API-сценаріїв тестування.

← Назад до меню

Мета

Навчитися створювати API-сценарії тестування (послідовності запитів), визначати контрольні точки та перевірки між кроками сценарію, аналізувати статус-коди і дані відповідей.

Навчальний ресурс

У роботі використовується “API Scenario Runner” — інструмент, який виконує готові сценарії. Студенти запускають сценарій, аналізують кожен крок та фіксують результати у звіті.

Відкрити API Scenario Runner (ЛР7)

Що потрібно зробити

  1. Відкрити Runner і прочитати “About / Scenario Rules”.
  2. Виконати мінімум 5 сценаріїв.
  3. Для кожного сценарію заповнити таблицю: крок → метод → endpoint → очікування → фактичний результат.
  4. Зробити скріншоти виконання (мінімум 6).
  5. Оформити звіт і висновок.

Що здати

  • Скріншоти 6+
