Мета

Навчитися знаходити дефекти у веб-застосунку та оформлювати баг-репорти за стандартною структурою з указанням severity/priority і доказами (скріншоти).

Навчальний сайт для тестування

У даній лабораторній використовується сайт “Bug Hunt Demo”, який містить навмисно додані помилки для навчального тестування.

Відкрити сайт для тестування (ЛР5)

Що потрібно зробити

  1. Відкрити Test Target — ЛР5 та прочитати блок “About / Requirements”.
  2. Виконати тестування основних функцій сайту (форма входу, калькулятор, профіль, пошук).
  3. Виявити мінімум 5 дефектів та зафіксувати їх у вигляді баг-репортів.
  4. Для кожного баг-репорту додати докази (скріншот або серію скріншотів).
  5. Оформити звіт з переліком знайдених дефектів.

Що здати

  • Звіт ЛР5 (Google Docs / DOCX / PDF)
  • Мінімум 5 баг-репортів
  • Скріншоти-докази для кожного дефекту
