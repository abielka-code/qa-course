Лабораторна робота №14

Аналіз логів та збір доказів дефекту (DevTools)

Опис роботи

У цій лабораторній роботі студент працює як тестувальник та навчається знаходити причини дефектів через DevTools: вкладки Console і Network, а також правильно збирати докази (скріншоти помилок, запитів, відповідей, статус-кодів).

Що потрібно зробити

  • Виконати ручне тестування target-сайту.
  • Проаналізувати Console (errors/warnings) після дій користувача.
  • Проаналізувати Network (запити, статуси, Response, повторні запити).
  • Знайти дефекти, оформити баг-репорти з доказами (Console/Network).
  • Підготувати Test Summary Report з рішенням Go/No-Go.
Відкрити target (L14) Повернутися на головну

Артефакти, які потрібно здати

  • Чек-лист (мінімум 15–25 пунктів).
  • Тест-кейси (мінімум 6–10, з акцентом на DevTools докази).
  • Баг-репорти (мінімум 4–6 або всі знайдені).
  • Підсумковий звіт (Test Summary Report).
  • Докази: скріншоти Console + Network (Headers/Response/Status).
